SEMANTIC HTML

Building meaningful web structures

Why Semantic HTML?

Semantic tags tell browsers and screen readers what your content means, not just how it looks.

üìö Analogy: If HTML were a book:
- <div> = Blank pages
- Semantic tags = Chapter titles, paragraphs, footnotes
semantic-page.html 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Semantic Page</title>
  <style>
    body { font-family: Arial; margin: 0; }
    header { background: #6e45e2; color: white; padding: 1rem; }
    nav { background: #0f0e17; padding: 1rem; }
    nav a { color: white; margin-right: 1rem; text-decoration: none; }
    main { padding: 2rem; max-width: 800px; margin: 0 auto; }
    section { margin-bottom: 2rem; }
    article { background: #f5f5f5; padding: 1rem; border-radius: 4px; }
    aside { background: #e9e9e9; padding: 1rem; border-radius: 4px; }
    footer { background: #0f0e17; color: white; padding: 1rem; text-align: center; }
  </style>
</head>
<body>
  <header>
    <h1>Web Dev School</h1>
    <nav>
      <a href="#">Home</a>
      <a href="#">Courses</a>
      <a href="#">About</a>
    </nav>
  </header>
  
  <main>
    <section>
      <h2>Latest Articles</h2>
      <article>
        <h3>Understanding Semantic HTML</h3>
        <p>Semantic HTML helps search engines and screen readers understand your content structure.</p>
        <time datetime="2023-06-15">June 15, 2023</time>
      </article>
    </section>
    
    <aside>
      <h3>Related Links</h3>
      <ul>
        <li><a href="#">HTML5 Specification</a></li>
        <li><a href="#">Accessibility Guide</a></li>
      </ul>
    </aside>
  </main>
  
  <footer>
    <p>© 2023 Web Dev School. All rights reserved.</p>
    <address>Contact us at: info@webdev.school</address>
  </footer>
</body>
</html>

1. Main Structural Tags

<header> <!-- Top section (logo/navigation) --> <nav> <!-- Navigation links --> <main> <!-- Primary content --> <section> <!-- Thematic grouping --> <article> <!-- Self-contained content --> <aside> <!-- Side content --> <footer> <!-- Bottom section -->
üí° Tip: These tags work exactly like divs, but with meaning!

2. Why It Matters

  • Accessibility: Screen readers understand your structure
  • SEO: Search engines prioritize semantic content
  • Maintenance: Easier for developers to understand

3. Semantic Text Tags

<time datetime="2023-01-01">New Year</time> <address>contact@example.com</address> <figure> <img src="photo.jpg" alt="Description"> <figcaption>A caption</figcaption> </figure> <mark>Highlighted text</mark>
