Accessibility Best Practices

Semantic HTML

Use proper HTML elements for their intended purpose.

Good: <button>Submit</button>
Bad: <div onclick="submitForm()">Submit</div>

Text Alternatives

Always provide text alternatives for non-text content.

Good: <img src="chart.jpg" alt="Sales growth chart showing 20% increase">
Bad: <img src="chart.jpg">

Keyboard Navigation

Ensure all functionality is available via keyboard.

Good: <a href="#">Link</a>
<button>Button</button>
Bad: <div tabindex="0">Fake button</div>

ARIA Attributes

Use ARIA to enhance accessibility when needed.

Good: <nav aria-label="Main navigation">
Bad: <div class="nav">

Color Contrast

Ensure sufficient contrast between text and background.

Good: color: #000; background: #fff;
Bad: color: #999; background: #eee;

Form Accessibility

Always associate labels with form controls.

Good: <label for="email">Email:</label>
<input id="email" type="email">
Bad: <div>Email:</div>
<input type="email">
